# Adaptive Interview Question Seeds

When user memory shows missing metrics, ask questions that force measurable impact: "Tell me about a decision where you improved a metric and how you knew it worked." When memory shows weak technical depth, ask for architecture, tradeoffs, tests, monitoring, and rollout details.

For backend candidates, useful adaptive seeds include API reliability, data consistency, cache invalidation, queue failure handling, observability, database migration, and incident response. For frontend candidates, use accessibility, performance, state management, user feedback, design collaboration, and experimentation.

For senior candidates, prefer questions that combine ambiguity, stakeholder tradeoffs, scale, and long-term ownership. For junior candidates, prefer debugging, learning quickly, validation, and communication with teammates.

If the target company is impact-heavy, ask for metrics and ownership. If the company is quality-heavy, ask for edge cases, maintainability, and customer risk.
